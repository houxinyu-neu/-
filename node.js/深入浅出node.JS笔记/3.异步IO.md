# 异步I/O

异步I/O、事件驱动、单线程三个组成了`node.JS`的基调。

## 异步I/O的原因

1. 用户体验角度：超过100ms的脚本执行时间将会造成卡顿。

   + 同步I/O的时间复杂度：O（n）=（m+n）
   + 异步I/O的时间复杂度：O（n）= Max（m，n）

2. 资源分配

   当前解决多任务执行的方法的不足

   + 单线程串行执行：代码阻塞造成性能问题。
   + 多线程并行执行：锁的问题、状态的同步。

   基于以上原因`node.JS`提出单线程异步I/O的方式。
